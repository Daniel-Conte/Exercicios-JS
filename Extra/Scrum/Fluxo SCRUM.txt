-> Resumo das etapas que o Scrum percorre durante um projeto

-> O projeto surge a partir de uma necessidade
    - Ex: O diretor de uma escola quer um sistema escolar para economizar tempo e papelada

-> Surge o projeto "Construção do sistema de gestão da escola Esperança"
    - Ainda não há detalhes sobre o projeto, é só uma ideia superficial

-> Fluxo do Scrum
    1. Visão
        - É criado o documento "Declaração de visão do projeto" para definir melhor a ideia e criar a base do projeto
            - É definido o objetivo do projeto, descrição, mapeamento de quem são as partes interessadas no projeto (todos que estão envolvidos) e outras informações importantes
            - É nesta etapa onde o Dono do Produto, Scrum Master e a Equipe Scrum serão definidos para desenvolver o projeto (no caso de empresas que possuem várias equipes)
            - Geralmente, todas as pessoas envolvidas no projeto irão participar da construção do documento de visão
            - Em relação a escrever o documento, não há um responsável exclusivo por fazer isto, geralmente será algum gestor/coordenador/líder de projetos
                - Se o Dono do Produto já estiver pré-definido antes da criação da visão, ele é quem escreverá o documento

    2. Backlog do Produto
        - Onde é listado todos os requisitos do produto a ser desenvolvido (Escopo)
            - Ex: O sistema deve registrar a nota de todos os alunos
            - No Scrum, cada requisito é chamado de História de usuário(User Story)
            - Histórias de usuário que são grandes e não cabem no desenvolvimento da Sprint são chamadas de "Épicos"
                - Épicos são divididos em histórias de usuário menores
            - O dono do produto é responsável por coletar as histórias de usuário com as partes interessadas

    3. Planejamento de Release
        - Onde será planejado as macro entregas do projeto (as entregas em que já é possível utilizar o produto desenvolvido), definindo o que será entregue e quando
            - Isso sana as expectativas de datas das partes envolvidas no projeto

    4. Backlog da Sprint
        - É o planejamento do que vai ser feitos na próxima Sprint
            - Ex: Será desenvolvido o cadastro de alunos no sistema
        - Note que não é feito o planejamento total do projeto, é planejado conforme o desenvolvimento
        - Quando definido, é mostrado para o cliente decidir se aprova ou não

    5. Sprint
        - Onde será desenvolvido tudo o que foi planejado no Backlog da Sprint
        - Cada Sprint tem duração de 2 a 4 semanas

        4.1. Daily Scrum
            - É uma reunião com a equipe realizada TODOS os dias durante o Sprint para discutir sobre o andamento do projeto
            - É realizada no início do expediente
            - Tem duração de 15 min(geralmente)
            - É realizada com toda a equipe de pé, pra ser mais rápido e prático
            - O foco da reunião é analizar o painel de tarefas com o andamento da Sprint(que está visível para toda a equipe)
            - O Scrum Master faz 3 perguntas para cada integrante da equipe:
                - O que foi feito ontem?
                - O que vai fazer hoje?
                - Existe algum impedimento para progredir na Sprint?
            - O Scrum Master vai fazer uma lista com todos os impedimentos da equipe e vai trabalhar pra resolvê-los logo após a reunião
            - O Scrum Master vai mostrar o gráfico BurnDown
                - É um gráfico que possui a relação 'Quantidade de tarefas' X 'Dias da Sprint', que mostra 2 linhas:
                    - Uma linha mostra o progresso que foi planejado para a Sprint
                    - A outra linha mostra o progresso real da Sprint

    6. Reunião de Revisão da Sprint
        - Após cada Sprint, o produto desenvolvido é mostrado ao dono do produto e para as partes interessadas para avaliá-lo e dar feedback
            - A Equipe e o Scrum Master fazem a apresentação do que foi feito
            - Dura em torno de 4 horas(no caso de uma Sprint de 2 semanas)
        - Após a aprovação do cliente, o projeto é adaptado ao feedback fornecido e é planejado a novo Sprint (Backlog da Sprint)
        - Caso alguma história de usuário seja reprovada, ela volta para o Backlog do Produto

    7. Retrospectiva da Sprint
        - Tem objetivo de verificar necessidades e adaptações no processo de trabalho da equipe
        - Tem duração de 1,5 horas(no caso de uma Sprint de 2 semanas)
        - É uma reunião de "lições aprendidas"
        - Vai ser mostrado o que funcionou e o que não funcionou, o que deu e não deu certo...
        - É feita pela equipe Scrum
        - É discutido as inconsistências que ocorreram na Sprint, alguma ferramenta que não foi de boa escolha, algum membro não foi produtivo, o Scrum Master não engajou muito...
        - É traçado um plano para não cometer os erros novamente

    8. Release
        - Toda Sprint é uma nova versão do projeto, mas nem todas elas são versões prontas para o cliente já utilizar(releases)
            - Cada release é entregue geralmente com um intervalo de algumas Sprints

    9. Retrospectiva do Projeto
        - É basicamente uma retrospectiva da Sprint, só que realizada no final do projeto e com toda a equipe e partes interessadas referentes à gestão do projeto
        - Discutindo todos os erros e acertos durante o projeto e sugerindo o que deve mudar e o que deve continuar nos próximos projetos
